# Shortcutype user guide

## Start locally

Install Node.js 20 or newer. From a fresh checkout, run one command:

```bash
npm start
```

If dependencies are missing, the command installs the lockfile exactly and then starts the local app. Open the URL printed in the terminal.

## First run

Shortcutype asks a question by showing an action. Answer by pressing the real shortcut; do not type the action name.

The introduction has three steps:

1. See and press a visible answer.
2. Follow the same pattern for another action.
3. Recall the first answer with the keys hidden.

Choose the tools you use after the tutorial. Shortcutype immediately creates a relevant first practice set. The introduction can be skipped or replayed from Practice setup and the command palette.

## Practice controls

- Press `Enter` on Ready to continue the configured practice.
- Press the requested shortcut to answer.
- Press `F1` to reveal the answer. Revealed answers are assisted and unscored.
- Press `Ctrl + →` to skip.
- Press `Esc` or `Ctrl/Cmd + Shift + P` to open commands and pause timing.
- After results, press `Enter` to repeat.

If the requested answer itself is `Esc` or `Ctrl/Cmd + Shift + P`, it is treated as the answer instead of opening application commands.

## Practice choices

Ready offers direct goals: continue, five-minute warm-up, weak review, or practice a selected tool. Practice setup contains fixed/timed sessions, categories, specialties, learning/recall mode, platform, theme, language, sound, motion, and optional unscored system cards.

## Progress and privacy

Shortcutype is local-only. Progress, settings, and tool choices stay in the current browser profile. If storage is blocked, practice continues and the app clearly warns that progress will not be saved.

No account, server, telemetry, or analytics are used.
